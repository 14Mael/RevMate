# 成员 4 — 前端（Vue3 + Element Plus）

> 任务级计划，不含代码。你调所有人的接口，最怕等后端 —— 全程用 mock 先行。

**目标**：做出登录、资料管理、问答、出题四个页面，能完整演示核心闭环。

**依赖**：Plan 0 的接口契约（拿到契约就能用 mock 数据开工，不等后端）。

---

## 任务 1：前端工程与基础设施

- 在 `frontend/` 建 Vue3 工程（Vite），装 Element Plus、axios、vue-router、pinia
- 配 axios：base URL、请求拦截器自动带 `Authorization: Bearer <token>`、响应拦截器解包 `Result`、401 跳登录
- 配路由 + 登录态守卫
- **验收**：工程能启动；未登录访问受保护页跳转登录

## 任务 2：登录/注册页

- 表单（用户名/密码）、调注册与登录、token 存 pinia/localStorage
- 先指向 mock，交接点 A 后接真实接口
- **验收**：能登录拿到 token 并进入主界面

## 任务 3：资料管理页

- 文件上传组件（支持 txt/pdf/word/图片）、上传进度、资料列表（文件名/类型/状态）、删除
- 先用 mock 列表，交接点 A 后接真实接口
- **验收**：能上传、看到列表、删除

## 任务 4：问答页（核心演示页）

- 聊天式界面：输入问题、展示回答、**展示来源**（资料出处 / 网页链接）
- 区分"来自资料"和"来自联网"的来源样式
- 先用 mock 回答，交接点 B 后接真实 `/api/chat`
- **验收**：提问能显示回答和出处

## 任务 5：出题/答题页

- 选择资料 + 题型 + 数量 → 出题；渲染单选/填空/简答；可作答并对照答案/解析
- 先用 mock 题目，后接真实 `/api/quiz`
- **验收**：能生成并展示题目、能看到答案解析
