# 成员 2 — RAG 核心与多模态提取器（后端）

> 任务级计划，不含代码。你是技术核心，体现 Spring AI 的主要亮点。

**目标**：把资料提取为文本→切片→向量化入库；实现按用户隔离的检索；实现 RAG 问答 + 联网兜底。

**依赖**：Plan 0（骨架、契约、AI 配置）。成员1 的上传与「当前用户 id」（交接点 A 前可用本地写死文件解耦）。

**对外提供（别人依赖你的）**：
- `ingest(userId, materialId, file)`：解析+切片+向量化入库（供成员1 上传后调用）
- `retrieve(userId, query, topK)`：按用户过滤的相似度检索（供问答流程使用）
- `getMaterialContext(userId, materialId, maxChunks)`：按资料取代表性切片上下文（供成员3 出题复用）
- `removeByMaterial(materialId)`：删除资料时清理向量（供成员1 调用）
- `POST /api/chat` 的问答能力

---

## 任务 1：AI 配置与连通性

- 配好 DashScope：ChatModel、EmbeddingModel（text-embedding-v3）、ChatClient
- 写一个临时连通性测试：发一句话能拿到回复、能拿到 embedding 向量
- **验收**：能成功调用通义对话和 embedding

## 任务 2：提取器架构（可插拔）

- 定义 `ContentExtractor` 接口（supports + extract）
- 实现 `TikaExtractor`（txt/pdf/word）
- 实现 `ImageExtractor`（图片，调用 qwen-vl 识别图中文字/内容）
- 录音 `AudioExtractor`（paraformer）**列为加分项**，先不做
- `MaterialService` 按文件类型选择匹配的提取器
- **验收**：丢一个 PDF 和一张图片，分别能提取出文本

## 任务 3：切片与向量入库

- 用 TokenTextSplitter 之类把文本切片（带重叠）
- 每个切片带 metadata `{userId, materialId, source}` 写入 VectorStore（先用文件版 SimpleVectorStore）
- 暴露 `ingest(...)`、`getMaterialContext(...)` 和 `removeByMaterial(...)`
- **验收**：入库后能在向量库查到带正确 metadata 的切片

## 任务 4：按用户隔离的检索

- 实现 `retrieve(userId, query, topK)`：embedding 检索 + 按 userId 过滤 metadata
- 实现 `getMaterialContext(userId, materialId, maxChunks)`：按 userId + materialId 过滤 metadata，返回该资料的代表性切片文本
- **验收**：A 用户检索不到 B 用户的切片

## 任务 5：RAG 问答 + 联网兜底

- `POST /api/chat`：先 `retrieve`，命中则组 prompt（资料片段+问题）让 ChatClient 作答
- 按 Plan 0 契约组装结构化 `sources`（`{type, title, snippet, url?, materialId?, page?}`）：
  - 资料命中：`type=material`，带 `materialId`、`page`（有则填）、`snippet`（原文片段）
  - 命中不足（空或相关度低）：调用通义 `enable_search` 联网作答，来源 `type=web` 且**必带 `url`**
- **验收**：问资料里有的 → 返回 `material` 类来源（带 materialId/原文片段）；问资料里没有的 → 返回 `web` 类来源（带网页 url）
