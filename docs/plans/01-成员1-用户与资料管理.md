# 成员 1 — 用户与资料管理（后端）

> 任务级计划，不含代码。你是全队地基，2/3/4 都依赖你，**要最先出活**。

**目标**：完成登录注册（JWT）、文件上传、资料列表/删除，建好 users/materials 表。

**依赖**：Plan 0（骨架、契约、Result、Security 骨架、schema）。

**对外提供（别人依赖你的）**：
- 可用的 `POST /api/auth/register`、`POST /api/auth/login`
- 登录后的 JWT 鉴权（请求头解析 + 注入当前用户）
- `POST /api/materials` 上传后落库并返回 `materialId`
- 一个能拿到「当前登录用户 id」的方式，供成员2 在入库/检索时做用户隔离

---

## 任务 1：用户实体与注册

- 建 `User` 实体、Repository
- 实现注册：用户名查重、密码 BCrypt 加密入库
- **验收**：重复用户名报友好错误；数据库能看到加密后的密码

## 任务 2：登录与 JWT 签发

- 校验用户名密码，签发 JWT（含 userId、用户名、过期时间）
- 引入 jjwt，密钥放配置
- **验收**：正确密码返回 token，错误密码返回统一 401

## 任务 3：JWT 鉴权过滤器

- 实现过滤器：解析 `Authorization` 头，验签，放入 SecurityContext
- 提供「获取当前用户 id」的工具（供其他模块复用）
- 接入 Plan 0 的 Security 放行规则
- **验收**：带有效 token 能访问受保护接口；无/错 token 返回 401

## 任务 4：文件上传

- 实现 `POST /api/materials`：接收 multipart 文件
- 校验类型（txt/pdf/word/image，audio 加分项）和大小上限
- 把文件存到本地目录（如 `uploads/{userId}/`），在 materials 表记录元信息，status 初始为「处理中」
- **预留对接**：上传成功后需要触发成员2 的「解析+入库」——先留一个 Service 接口占位（交接点 A 时对接），自己阶段可先不真正解析
- **验收**：上传后文件落盘、materials 表有记录、返回 materialId

## 任务 5：资料列表与删除

- `GET /api/materials`：只返回当前用户的资料
- `DELETE /api/materials/{id}`：校验归属，删库记录 + 删文件（向量库清理由成员2 提供方法，交接点对接）
- **验收**：A 用户看不到 B 用户的资料；删除后列表消失

- 第 4~ ：任务 5 + 配合联调
