# Plan 0 — 项目骨架与接口契约（全员 Day 1 共建）

> 这是所有人的地基。Day 1 全员一起完成本计划，之后 4 人各自照成员计划并行开发。
> 本计划只描述"做什么、产出什么、验收标准"，不含具体代码（代码在实现阶段写）。

**目标**：搭好可运行的 Spring Boot 工程，定死全员共用的接口契约与数据格式，让 4 人能解耦并行。

**技术栈**：Spring Boot 3.x + JDK 21（LTS）+ spring-ai-alibaba-starter-dashscope 1.0.0.1 + MySQL + Vue3/Element Plus（前端独立目录）。

## 全局约束（所有任务都默认遵守）

- JDK 21（LTS）——spring-ai-alibaba 最低要求 17，这里统一用 21；**4 人必须装同一版本**
- 后端包名统一前缀：`com.team.study`（可改，定了就别变）
- 所有接口走 `/api/**`，统一返回包装 `Result<T>`：`{ code, message, data }`
- 所有需要登录的接口走 JWT，请求头 `Authorization: Bearer <token>`
- 前端工程放独立目录 `frontend/`，与后端物理隔离
- DashScope API Key 放配置文件，不硬编码、不提交进 git

## 统一开发环境（4 人必须一致）

Day 1 全员对齐以下版本，避免"我这能跑你那报错"：

| 工具 | 版本 | 说明 |
|---|---|---|
| JDK | **21**（LTS） | 统一发行版建议都用 Temurin/Adoptium |
| 构建 | Maven 3.9+ | 用工程自带的 `mvnw` 最稳，免装版本不一致 |
| Spring Boot | 3.x（最新稳定 3.x） | 由骨架统一锁定，别各自改 |
| spring-ai-alibaba | starter-dashscope **1.0.0.1** | 稳定版，别混用里程碑版 |
| MySQL | 8.x | 字符集 utf8mb4 |
| Node.js | 20 LTS（前端） | 配套 npm 10；前端 `frontend/` 目录用 |
| IDE | IDEA（建议） | 统一 `.editorconfig`，避免格式/编码差异 |

> API Key、数据库密码等放各自本地的 `application-local.yml` 并加入 `.gitignore`；仓库里只提交 `application.yml` 模板（占位）。

## 任务 1：创建可运行的 Spring Boot 骨架

**产出**：能 `mvn spring-boot:run` 启动的空工程。

- 用 Spring Initializr 建工程（Maven、Java 21、Spring Boot 3.x）
- 依赖：Web、Spring Security、MyBatis 或 JPA（团队选一种，建议 JPA 上手快）、MySQL Driver、Lombok、spring-ai-alibaba-starter-dashscope
- 建好包结构：`controller / service / repository(或 mapper) / entity / dto / config / ai / extractor`
- 写最简 `application.yml`：端口、数据源占位、dashscope.api-key 占位
- **验收**：启动不报错；访问一个临时 `/api/ping` 返回 `pong`
- 推到 git 远程仓库，建好 `main` 分支

## 任务 2：定死接口契约（最重要）

**产出**：一份 `API契约.md` + 对应的 DTO 类（只定义字段，不写逻辑）。

- 用 Apidog/Postman 建好接口集合，逐个写清路径、入参、返回 JSON：
  - `POST /api/auth/register` `{username, password}` → `Result<void>`
  - `POST /api/auth/login` `{username, password}` → `Result<{token}>`
  - `POST /api/materials`（multipart 文件）→ `Result<{materialId, filename, type, status}>`
  - `GET /api/materials` → `Result<[{id, filename, type, status, createdAt}]>`
  - `DELETE /api/materials/{id}` → `Result<void>`
  - `POST /api/chat` `{question}` → `Result<{answer, sources:[{source, snippet}]}>`
  - `POST /api/quiz` `{materialId, type, count}` → `Result<{questions:[{stem, options?, answer, analysis}]}>`
- 定下 `type` 取值：`single`(单选)/`fill`(填空)/`qa`(简答)
- 定下 `material.type` 取值：`txt/pdf/word/image/audio`
- **验收**：4 人都确认契约；DTO 类编译通过；契约文件入库

## 任务 3：统一基础设施（横切）

**产出**：全员共用的通用组件。

- `Result<T>` 统一返回包装 + 统一异常处理（全局 `@RestControllerAdvice`）
- CORS 配置（允许前端开发地址跨域）
- Spring Security 基础放行规则（`/api/auth/**`、`/api/ping`、swagger 放行，其余需登录）—— 具体 JWT 校验由成员1 实现，这里只搭骨架
- **验收**：未登录访问受保护接口返回统一的 401 包装；前端能跨域调通 `/api/ping`

## 任务 4：数据库与初始化脚本

**产出**：`schema.sql`（users、materials 两张表）。

- 按设计文档第 3 节建表字段
- 提供本地建库说明（库名、字符集 utf8mb4）
- **验收**：在本地 MySQL 跑通脚本；工程能连上库启动
